---
title: "Update Commission"
api: "PUT /v1/commissions/{id}"
description: "Update a commission's status, transaction status, or hold period"
---

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the commission to update
</ParamField>

## Body Parameters

All body parameters are optional. Only include the fields you want to update.

<ParamField body="status" type="string">
  Update the commission status. Valid values: `pending`, `pending_manual_approval`, `ready_for_payment`, `processing_payout`, `paid`, `declined`, `refunded`, `dispute`. Status changes may trigger webhook events (`transaction.approved`, `transaction.rejected`, `transaction.paid`).
</ParamField>

<ParamField body="transactionStatus" type="string">
  Update the transaction status. Valid values: `open`, `complete`, `trialing`, `failed`, `refunded`, `partial_refunded`. If set to `complete`, `userHasPaid` will be automatically set to `true`.
</ParamField>

<ParamField body="holdPeriodDays" type="integer">
  Update the hold period in days before the commission becomes available for payout. Must be a non-negative integer.
</ParamField>

## Response

The response includes the updated commission object with the same structure as the get endpoint.
