---
title: "Update Commission"
api: "PUT /v1/commissions/{id}"
description: "Update a commission's status, transaction status, or hold period"
---

## Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the commission to update
</ParamField>

## Body Parameters

All body parameters are optional. Only include the fields you want to update.

<ParamField body="status" type="string">
  Update the commission status. Valid values: `pending`, `pending_manual_approval`, `ready_for_payment`, `paid`, `declined`. Status changes may trigger webhook events (`transaction.approved`, `transaction.rejected`, `transaction.paid`).
</ParamField>

<ParamField body="sales_status" type="string">
  Update the sales/transaction status. Valid values: `open`, `complete`, `trialing`, `failed`, `refunded`, `partial_refunded`.
</ParamField>

<ParamField body="amount" type="number">
  Update the transaction amount. Must be a positive number.
</ParamField>

<ParamField body="commission_amount" type="number">
  Update the commission amount. Must be a non-negative number.
</ParamField>

<ParamField body="payment_intent_id" type="string">
  Update the payment intent ID. Set to `null` to clear.
</ParamField>

<ParamField body="invoice_id" type="string">
  Update the invoice ID. Set to `null` to clear.
</ParamField>

## Response

The response includes the updated commission object with the same structure as the get endpoint.

<RequestExample>
```bash cURL

  -H "Authorization: Bearer sk_live_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "paid"
  }'
```
</RequestExample>
