## Overview

Payout events track the payment lifecycle for affiliate commissions. Use these webhooks to sync payout data with your accounting system, notify affiliates about payment status, or integrate with payment providers.

<Note>
  All payout events except `payout.created` and `payout.deleted` automatically trigger a `payout.updated` event as well, giving you a universal event for all changes.
</Note>

## Events

<AccordionGroup>
  <Accordion title="payout.created" icon="plus">
    Triggered when a new payout is created for an affiliate. This happens when commissions are batched for payment.

    ```json
    {
      "id": "evt_abc123",
      "type": "payout.created",
      "createdAt": "2024-01-15T10:30:00.000Z",
      "data": {
        "payoutId": "pay_xyz789",
        "affiliateId": "aff_abc123",
        "affiliateProgramId": "prog_def456",
        "invoiceNumber": 1042,
        "amount": 250.00,
        "status": "PENDING",
        "isManagedPayout": false,
        "paidAt": null,
        "createdAt": "2024-01-15T10:30:00.000Z",
        "updatedAt": "2024-01-15T10:30:00.000Z"
      }
    }
    ```
  </Accordion>

  <Accordion title="payout.updated" icon="arrows-rotate">
    Triggered whenever a payout is modified. This is a universal event that fires alongside other specific events (paid, failed).

    ```json
    {
      "id": "evt_abc123",
      "type": "payout.updated",
      "createdAt": "2024-01-15T10:30:00.000Z",
      "data": {
        "payoutId": "pay_xyz789",
        "affiliateId": "aff_abc123",
        "affiliateProgramId": "prog_def456",
        "invoiceNumber": 1042,
        "amount": 250.00,
        "status": "PROCESSING",
        "isManagedPayout": true,
        "paidAt": null,
        "createdAt": "2024-01-15T10:30:00.000Z",
        "updatedAt": "2024-01-15T12:00:00.000Z"
      }
    }
    ```
  </Accordion>

  <Accordion title="payout.paid" icon="money-bill-transfer">
    Triggered when a payout is marked as completed and the affiliate has been paid.

    ```json
    {
      "id": "evt_abc123",
      "type": "payout.paid",
      "createdAt": "2024-01-15T10:30:00.000Z",
      "data": {
        "payoutId": "pay_xyz789",
        "affiliateId": "aff_abc123",
        "affiliateProgramId": "prog_def456",
        "invoiceNumber": 1042,
        "amount": 250.00,
        "status": "COMPLETED",
        "isManagedPayout": true,
        "paidAt": "2024-01-15T14:00:00.000Z",
        "createdAt": "2024-01-15T10:30:00.000Z",
        "updatedAt": "2024-01-15T14:00:00.000Z"
      }
    }
    ```
  </Accordion>

  <Accordion title="payout.failed" icon="circle-xmark">
    Triggered when a payout fails. This can happen due to payment provider errors, invalid payment details, or insufficient funds.

    ```json
    {
      "id": "evt_abc123",
      "type": "payout.failed",
      "createdAt": "2024-01-15T10:30:00.000Z",
      "data": {
        "payoutId": "pay_xyz789",
        "affiliateId": "aff_abc123",
        "affiliateProgramId": "prog_def456",
        "invoiceNumber": 1042,
        "amount": 250.00,
        "status": "FAILED",
        "isManagedPayout": true,
        "paidAt": null,
        "createdAt": "2024-01-15T10:30:00.000Z",
        "updatedAt": "2024-01-15T14:00:00.000Z"
      }
    }
    ```
  </Accordion>

  <Accordion title="payout.deleted" icon="trash">
    Triggered when a payout is deleted from the system.

    ```json
    {
      "id": "evt_abc123",
      "type": "payout.deleted",
      "createdAt": "2024-01-15T10:30:00.000Z",
      "data": {
        "payoutId": "pay_xyz789",
        "affiliateId": "aff_abc123",
        "affiliateProgramId": "prog_def456",
        "invoiceNumber": 1042
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Payload Fields

| Field | Type | Description |
|-------|------|-------------|
| `payoutId` | string | Unique identifier for the payout |
| `affiliateId` | string | The affiliate receiving this payout |
| `affiliateProgramId` | string | The program this payout belongs to |
| `invoiceNumber` | number | Sequential invoice number for the payout |
| `amount` | number | Total payout amount |
| `status` | string | Payout status (see below) |
| `isManagedPayout` | boolean | Whether this is an auto-managed payout |
| `paidAt` | string \| null | ISO 8601 timestamp when paid |
| `createdAt` | string | ISO 8601 timestamp |
| `updatedAt` | string | ISO 8601 timestamp |

## Payout Status Values

| Status | Description |
|--------|-------------|
| `PENDING` | Payout created, awaiting processing |
| `PROCESSING` | Payout is being processed |
| `COMPLETED` | Payout successfully completed |
| `FAILED` | Payout failed |
